CRITICAL FIX - Missing Route for Property Update
═══════════════════════════════════════════════════════════

## The Problem You Found

404 Error when trying to edit a property:
```
POST https://2clv.com/jrk/api/properties-update
404 (Not Found)
```

## Root Cause

The `properties-update.php` file existed in the codebase, but the
router in `index.php` didn't have a route registered for it!

**Missing from index.php:**
```php
$router->post('/api/properties-update', __DIR__ . '/api/properties-update.php');
```

Without this route, all requests to `/api/properties-update` would
return 404 even though the file exists.

## The Fix

Added the missing route to `index.php` line 58:

**Before:**
```php
// Property routes
$router->get('/api/properties', __DIR__ . '/api/properties.php');
$router->get('/api/properties-list', __DIR__ . '/api/properties-list.php');
$router->post('/api/properties-create', __DIR__ . '/api/properties-create.php');
$router->post('/api/properties-delete', __DIR__ . '/api/properties-delete.php');
```

**After:**
```php
// Property routes
$router->get('/api/properties', __DIR__ . '/api/properties.php');
$router->get('/api/properties-list', __DIR__ . '/api/properties-list.php');
$router->post('/api/properties-create', __DIR__ . '/api/properties-create.php');
$router->post('/api/properties-update', __DIR__ . '/api/properties-update.php');  ← NEW
$router->post('/api/properties-delete', __DIR__ . '/api/properties-delete.php');
```
